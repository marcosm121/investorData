# Design: Markdown Summaries for GET /news

**Date:** 2026-04-05
**Status:** Approved

## Overview

Extend the existing `GET /news` endpoint to include a `summary` field per article. Summaries are generated by Groq LLM from the real article content, extracted via Jina Reader API (`https://r.jina.ai/{url}`). Jina failures fall back to NewsAPI title + description. Results are cached alongside the existing 2h cache.

## Architecture

Two new services, no new endpoints.

```
src/
  services/
    jinaService.ts     ← fetches article content via Jina Reader, handles fallback
  (groqService.ts)     ← add summarizeArticle() function
  (newsController.ts)  ← orchestrate content fetch + summarization after selectTopArticles()
```

## Extended Data Flow

```
GET /news
  → [cache hit] return cached response immediately
  → fetchAllNews() → filter SOURCE_BLACKLIST → dedup
  → selectTopArticles() → 5 articles
  → fetchContent() × 5 in parallel       ← new (jinaService)
      • domain in JINA_DOMAIN_BLACKLIST → skip Jina, use fallback directly
      • Jina success + len >= 200 chars + no "blocked" message → use content
      • Jina fails / blocked / short → fallback: title + ". " + description
  → summarizeArticle() × 5 in parallel   ← new (groqService)
  → articles with `summary` field added
  → store in cache
  → return response
```

## Blacklists

```typescript
// Filter applied after fetchAllNews(), before selectTopArticles()
const SOURCE_BLACKLIST = ['Yahoo Entertainment'];

// Filter applied in jinaService before calling Jina
const JINA_DOMAIN_BLACKLIST = ['apnews.com'];
```

Both are hardcoded arrays — easy to extend.

## Response Shape

```json
[
  {
    "title": "YPF anuncia contrato de GNL",
    "url": "https://...",
    "source": "Infobae",
    "publishedAt": "2026-04-05T10:00:00Z",
    "category": "watchlist",
    "summary": "### YPF avanza en exportaciones de GNL\n\nEn un acuerdo firmado..."
  }
]
```

## Jina Reader Integration

- URL pattern: `GET https://r.jina.ai/{article_url}`
- No API key required (free tier)
- Timeout: 8 seconds per request
- Fallback triggers when:
  - Request throws / times out
  - Response body contains `"blocked"` (case-insensitive)
  - Response body length < 200 characters

## Groq Summarization Prompt

```
Dado el siguiente contenido de un artículo periodístico, escribí un resumen
conciso en español en formato markdown (máximo 150 palabras).
Usá un título H3 y 2-3 párrafos breves.

Título: {title}
Contenido: {content}
```

- Model: same `GROQ_MODEL` env var
- Temperature: 0.3

## Source Blacklist Application

Applied in `newsService.fetchAllNews()` after dedup, before returning:

```typescript
.filter(article => !SOURCE_BLACKLIST.includes(article.source))
```

## Out of Scope

- Persisting summaries to MongoDB
- Per-article cache invalidation
- Dynamic blacklist management via API
